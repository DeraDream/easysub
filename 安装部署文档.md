# 省心订阅 EasySub —— NAS / Docker 安装部署文档

适用于 **TrueNAS SCALE、飞牛 fnOS、群晖 Synology、威联通 QNAP、Unraid** 以及任意支持 Docker 的 Linux。

---

## 0. 开始之前（务必先读）

- **本项目不自带数据库**，需要连接你**已有的 MySQL 8**。请先准备好一个 MySQL，并新建一个**空库**（如 `easysub`）专供本项目使用。
- **数据库连接信息不写在配置里**：容器启动后，首次访问网页会进入「**数据库安装向导**」，在网页里填主机 / 端口 / 用户 / 密码 / 库名，测试通过后保存初始化。
- 默认对外端口：**`8842`**（映射到容器内 `8000`，可自行修改）。
- 需要持久化的目录：容器内 **`/app/data`**（保存数据库连接配置 + 上传的图标）。务必挂载到 NAS 的持久化路径或命名卷，否则重启会丢失向导配置。
- 时区默认 `Asia/Shanghai`，可用环境变量 `TZ` 修改。

### 0.1 先在 MySQL 建一个空库（示例）
```sql
CREATE DATABASE easysub CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
-- 可选：单独建账号并授权
CREATE USER 'easysub'@'%' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON easysub.* TO 'easysub'@'%';
FLUSH PRIVILEGES;
```
> ⚠️ 安装向导里**主机不要填 `localhost`**。容器里的 localhost 是容器自己。
> 请填 MySQL 实际地址，例如 NAS 局域网 IP `192.168.1.10`，或同一 Compose 内 MySQL 服务名。

### 0.2 关键环境变量（可在各平台的 environment 里设置）
| 变量 | 说明 | 示例 |
|------|------|------|
| `JWT_SECRET` | 登录令牌密钥，**务必改成随机串** | `openssl rand -hex 32` 的输出 |
| `TZ` | 时区 | `Asia/Shanghai` |
| `TELEGRAM_BOT_TOKEN` | 可留空，后续在网页「设置」里按用户单独填 | |
| `REMINDER_SCAN_TIME` | 每日提醒扫描时间 | `09:00` |
| `ADMIN_USERNAME` / `ADMIN_PASSWORD` / `ADMIN_EMAIL` | 首个管理员（初始化时创建） | `admin` / 改成强密码 |
| `EXCHANGE_API_BASE` / `EXCHANGE_API_URL` | 汇率源，默认免费免 key | `USD` / `https://open.er-api.com/v6/latest/` |

## 1. 通用方式（命令行 Docker Compose，最稳）

适用于任何能 SSH 的 NAS / Linux。

```bash
# 1) 取得项目（git 或上传整个项目文件夹到 NAS）
git clone <你的仓库地址> easysub && cd easysub
#   或：把项目文件夹整体放到 NAS 共享目录，再 cd 进去

# 2) 准备配置
cp .env.example .env
nano .env           # 改 JWT_SECRET / ADMIN_PASSWORD 等

# 3) 启动（单容器，直接用 8842 端口访问；会自动构建镜像）
docker compose -f docker-compose.fnos.yml up -d --build

# 4) 浏览器访问
#    http://<NAS-IP>:8842  → 进入「数据库安装向导」填你现有 MySQL
```

查看日志 / 停止：
```bash
docker compose -f docker-compose.fnos.yml logs -f
docker compose -f docker-compose.fnos.yml down      # 不要加 -v，加 -v 会清空 ./data
```

> 想要**域名 + 自动 HTTPS**：改用根目录 `docker-compose.yml`（含 Caddy），并编辑 `Caddyfile` 把 `your-domain.com` 换成你的域名（DNS 需解析到本机）。见第 8 节。

## 2. Portainer（图形化 Stack，跨平台通用）

很多 NAS 都能装 Portainer，操作最统一。

1. Portainer → **Stacks → Add stack**。
2. 命名 `easysub`。
3. **Build method** 选 **Repository**（填你的 Git 仓库地址，Compose path 填 `docker-compose.fnos.yml`）；
   或选 **Web editor**，把 `docker-compose.fnos.yml` 内容粘贴进去（注意这种方式需要 NAS 上已有项目源码用于构建）。
4. 在 **Environment variables** 里加 `JWT_SECRET`、`ADMIN_PASSWORD` 等。
5. **Deploy the stack** → 访问 `http://<NAS-IP>:8842`。

## 3. TrueNAS SCALE

TrueNAS SCALE（24.10 Electric Eel 起为 Docker 引擎）支持自定义应用。

### 方法 A：Custom App（推荐，图形）
1. 准备持久化目录：**Datasets** 新建一个数据集，例如 `tank/apps/easysub`（用于 `/app/data`）。
2. **Apps → Discover Apps → 右上角 Custom App**。
3. 填写：
   - **Image repository**：先在能构建的机器上 `docker build` 出镜像并推送到镜像仓库（或 Docker Hub），这里填该镜像名；
   - 若不想自建镜像，用**方法 B** 的 compose（带 build 上下文）更省事。
4. **Port**：Container Port `8000` → Node Port `8842`。
5. **Environment Variables**：`JWT_SECRET`、`TZ`、`ADMIN_PASSWORD`…
6. **Storage**：Host Path `/mnt/tank/apps/easysub` → Mount Path `/app/data`。
7. 部署后访问 `http://<TrueNAS-IP>:8842`。

### 方法 B：YAML / Compose
新版「Custom App」支持直接粘贴 YAML。把项目放到 NAS 数据集后，使用以下内容（build 方式）：
```yaml
services:
  app:
    build: /mnt/tank/apps/easysub-src     # 指向放置项目源码的数据集
    container_name: easysub-app
    restart: unless-stopped
    environment:
      JWT_SECRET: 改成随机串
      TZ: Asia/Shanghai
      ADMIN_USERNAME: admin
      ADMIN_PASSWORD: 改成强密码
    volumes:
      - /mnt/tank/apps/easysub:/app/data
    ports:
      - "8842:8000"
```
> TrueNAS 部分版本对 build 支持有限，若构建失败，请在别的机器构建好镜像后改用方法 A 的镜像方式。

## 5. 群晖 Synology（Container Manager）

DSM 7.2+ 的 **Container Manager** 支持 Compose 项目。

1. 把项目文件夹上传到群晖共享文件夹，例如 `/volume1/docker/easysub`。
2. **Container Manager → 项目 → 新增**：
   - 项目名称 `easysub`；
   - 路径选 `/volume1/docker/easysub`；
   - 来源选「使用现有的 docker-compose.yml」→ 选择 **`docker-compose.fnos.yml`**（或把其内容粘到编辑器）。
3. 在 compose 的 `environment` 中改好 `JWT_SECRET` / `ADMIN_PASSWORD`。
4. 确认端口映射 `8842:8000`，卷 `./data` 会落在项目文件夹下的 `data`。
5. 构建并启动 → 访问 `http://<群晖IP>:8842`。

> 群晖自带的 MySQL/MariaDB 套件也可作为数据库；安装向导里主机填群晖局域网 IP（不要填 localhost），端口按套件实际（常见 3306）。

## 6. 威联通 QNAP（Container Station）

1. 将项目文件夹放到 `/share/Container/easysub`。
2. **Container Station → 应用程序（Applications）→ 创建 → 用 docker-compose**。
3. 粘贴 `docker-compose.fnos.yml` 内容，按需改 `environment` 与端口 `8842:8000`，卷指向 `/share/Container/easysub/data`。
4. 创建并启动 → 访问 `http://<QNAP-IP>:8842`。

## 7. Unraid

1. 安装社区插件 **Compose Manager**（Community Applications 里搜索）。
2. 新建一个 Compose Stack，粘贴 `docker-compose.fnos.yml`。
3. 卷 `/app/data` 映射到 `/mnt/user/appdata/easysub`，端口 `8842:8000`。
4. Compose Up → 访问 `http://<Unraid-IP>:8842`。

> 或者在别的机器构建镜像后，用 Unraid 的「Add Container」手动填镜像、端口 `8842:8000`、路径 `/mnt/user/appdata/easysub:/app/data`、环境变量。

## 8. 启用域名 + 自动 HTTPS（可选）

使用根目录 `docker-compose.yml`（包含 Caddy 反向代理）：

```bash
cp .env.example .env && nano .env        # 改密钥
nano Caddyfile                            # 把 your-domain.com 改成你的域名
docker compose up -d --build
```
- 域名 DNS 需解析到该 NAS 公网 IP，并放行 `80/443`。
- Caddy 会自动申请并续期证书，之后用 `https://你的域名` 访问。
- 本地无域名测试：编辑 `Caddyfile`，启用文件里 `:80 { ... }` 那段。

## 9. 首次使用流程

1. 浏览器打开 `http://<NAS-IP>:8842`（或你的域名）。
2. 进入「**数据库安装向导**」→ 填你现有 MySQL 的 主机 / 端口 / 用户 / 密码 / 库名 → 「**测试连接**」→ 「**保存并初始化**」。
3. 用 `.env`（或 environment）里的 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 登录。
4. 进入「设置」配置 Telegram（每个用户独立）、语言、主题、基准货币等。

## 10. 升级与备份

- **增量升级（不丢数据）**：`./update.sh` 或 `git pull && docker compose -f docker-compose.fnos.yml up -d --build`。
  数据在你的外部 MySQL 与 `data` 目录中，重建镜像不会清除；新增数据库列由 `backend/app/migrate.py` 启动时自动补齐。详见 **《升级与GitHub指南.md》**。
- **备份**：定期备份 ① 你的 MySQL 库；② 项目的 `data/`（含 `db_config.json` 与上传图标）。

## 11. 常见问题（FAQ）

| 现象 | 原因 / 解决 |
|------|------|
| 向导测试连接失败 | 主机别填 `localhost`；填 MySQL 真实 IP；确认 MySQL 允许远程（`bind-address` 与账号 `@'%'` 授权）；端口放行。 |
| 重启后又回到安装向导 | `/app/data` 没做持久化挂载，导致 `db_config.json` 丢失。检查卷映射。 |
| 端口冲突 | 改映射左侧端口，如 `9000:8000`，再用对应端口访问。 |
| 时间 / 提醒时间不对 | 设置 `TZ=Asia/Shanghai`，并确认 `REMINDER_SCAN_TIME`。 |
| 图标加载慢或缺失 | 首次会从公共 favicon 服务抓取并缓存到 `data/icons/library/`，需容器能访问外网。 |
| 升级后页面没变化 | 浏览器强刷（Ctrl/Cmd+Shift+R）；确认镜像已重建（`--build`）。 |
| 邮件验证码发不出 | 未配置 SMTP 时注册不需邮箱验证（仍需管理员审核）；要启用请配置 `SMTP_*`。 |
